# Implementation Patterns

## Pattern 1 - Synchronous request/response
Use for interactive user requests with immediate completion.

## Pattern 2 - Asynchronous completion
Use for long integrations or durable background work.

## Pattern 3 - Event-triggered capability
Use when a business event should invoke an agent capability.

## Pattern 4 - Human-in-the-loop
Use when an approval gate must occur before execution.

## Pattern 5 - Connector reuse
Use a shared integration service to avoid duplicating connector logic across executors.

## Pattern 6 - Runtime specialization
Use distinct executors when frameworks, security boundaries, or scaling profiles differ.
