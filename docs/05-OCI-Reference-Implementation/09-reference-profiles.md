# Reference Profiles

The logical architecture can be deployed using different OCI reference profiles.

## Profile A - OCI native

Use OCI managed services wherever possible and deploy platform logic on OKE only where necessary.

## Profile B - OCI plus third-party models

Use OCI for identity, network, security, observability, and execution while consuming external model endpoints where required by business policy.

## Profile C - Hybrid enterprise

Deploy the platform on OCI and connect to on-premises or other cloud enterprise systems through private connectivity.

## Profile D - Sovereign or dedicated deployment

Use the OCI region or deployment model that satisfies data residency, sovereignty, or regulatory requirements.

## Profile selection criteria

- Data residency
- Regulatory constraints
- Required integrations
- Model choice
- Operational ownership
- Recovery objectives
- Customer preference
