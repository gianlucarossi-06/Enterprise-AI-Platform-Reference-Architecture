# Observability Model

The observability model separates signal production from signal consumption.

## Signal producers

- AI Orchestrator
- AI Agent Gateway
- Agent Executors
- Tool Adapters
- AI Integration Service
- Enterprise Resource connectors

## Signal types

- traces for execution paths
- metrics for throughput, latency, retries, and saturation
- logs for structured events and errors
- audit events for security and compliance
- lineage events for prompt, tool, and capability usage

## Signal consumers

- dashboards
- alerts
- incident response workflows
- security operations
- compliance evidence stores
- cost and usage analytics
